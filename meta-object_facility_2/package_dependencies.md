# Зависимости между пакетами MOF

<img src="artifacts/packages.svg" width="600" alt="Зависимости между пакетами MOF">

С описанием каждого из пакетов можно ознакомиться в таблице ниже:

<img src="artifacts/packages_table.svg" width="600" alt="Описание пакетов MOF">

**MOF::Reflection** добавляет в метамодель универсальные операции (getMetaClass, container, get, set и т.д.). Считается, что когда определяется метамодель через MOF, эти операции становятся доступны для любых элементов модели. MOF::Reflection — это надстройка, которая даёт возможность самоописания.

**MOF::Extension**. Иногда нужно добавить к элементам модели временную или инструментальную информацию, которая не входит в основную метамодель. Например, пометка, что класс требует особой генерации кода. Вместо того чтобы менять метамодель, MOF::Extension предоставляет простые пары «имя-значение» (Tag), то есть добавляет механизм аннотаций.

**MOF::Common** предоставляет общие структуры данных (ReflectiveCollection, ReflectiveSequence) для работы с многозначными свойствами. Они нужны, чтобы единообразно обрабатывать коллекции и при рефлексии, и при идентификации, и при расширениях. Это утилитарный пакет.

**MOF::Identifiers** добавляет возможность задавать для элементов ID и контексты (Extent).

В UML нет рефлексии, идентификаторов и тегов, поэтому MOF добавляет их как отдельные пакеты.

Теперь в модели есть несколько независимых пакетов: Common, Reflection, Identifiers, Extension. Каждый добавляет свою функциональность, но не всегда нужна вся мощь. Например, для простых инструментов (EMF в Eclipse) достаточно только ядра (классы, атрибуты, ассоциации).
Для этих целей OMG создала **EMOF (Essential MOF)** - строгое подмножество, которое получается:

- слиянием (merge) пакетов Common, Reflection, Identifiers, Extension с ядром (Core);
- наложением ограничений на существующие в UML классы. Эти ограничения записаны в OCL (раздел 12.4 спецификации) и фактически разрешают только те конструкции, которые легко отобразить на объектно-ориентированные языки и XML.

**CMOF (Complete MOF)** - это EMOF и дополнительные возможности, которые снимают многие ограничения.

Принято считать, что CMOF используется для описания сложных метамоделей (UML, SysML, CWM), а EMOF - для простых.
