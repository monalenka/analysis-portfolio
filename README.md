## Ключевые компетенции
- **cбор и анализ требований** (функциональные / нефункциональные, user stories)
- **моделирование процессов** (BPMN, IDEF0, Activity diagrams)

## Кейсы и проекты

### Оптимизация процесса ликвидации академических задолженностей в ТГУ
> Учебный хакатон (совместный проект аналитиков и юристов)
**Задача:** проанализировать хаотичный процесс пересдач, выявить противоречия в документах, предложить целевую модель и передать требования юристам для написания нового регламента.

**Артефакты:**
- AS-IS и TO‑BE диаграммы активностей
- AS-IS и TO-BE RACI-матрицы
- документ с требованиями к регламенту
- итоговый локальный нормативный акт.

[Подробный разбор кейса](case_academic_debt/README.md)
