### Платформа монетизации обезличенных данных с согласия клиента
> Кейс-интенсив от Т-банка и ТПУ (команда из 2 человек)

**Задача:** разработать концепцию платформы, где клиенты банка добровольно делятся анонимизированными данными за вознаграждение, а компании получают агрегированные аналитические отчёты.

**Артефакты:**
- use case диаграммы (клиент и компания)
- 5 диаграмм последовательностей (согласие, расчёт дохода, вывод средств, отзыв согласия и др.)
- прототипы интерфейсов клиентской и B2B-части
- модель монетизации (ценообразование, распределение долей)
- анализ конкурентов, риски, метрики успеха.

[Подробный разбор кейса case_monetization_of_data](case_monetization_of_data)

### TravelTech: резервный сценарий для ML-рекомендаций
> Кейс от Т-банка

**Задача:** спроектировать резервный механизм для ML-сервиса персонализированных рекомендаций отелей/активностей/маршрутов.

**Артефакты:**
- BPMN-диаграмма процесса запроса (с таймаутом и fallback-веткой)
- State Machine Diagram для Circuit Breaker
- Sequence Diagram для двух сценариев
- таблица решений логики Fallback Engine
- таблица нефункциональных требований (таймауты, пороги, время восстановления)
