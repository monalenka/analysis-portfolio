# Дополнительные артефакты

В этой папке собраны небольшие самостоятельные работы (зачастую выполненные в рамках учебных работ), которые не вошли в основные кейсы, но демонстрируют мои знания в смежных областях.

## Нефункциональные требования к цифровой платформе обучения программированию

Требования, сформулированные по методологии Planguage (Том Гилб), включают 16 атрибутов качества: доступность, совместимость, устойчивость, безопасность, эффективность, модифицируемость, повторное использование, удобство использования, установки, производительность, защита, тестируемость, целостность, переносимость, масштабируемость и надёжность.
Каждое требование содержит измеримые метрики с уровнями Goal, Stretch, Wish.

[Нефункциональные требования](NFR_Planguage.md)
